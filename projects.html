In Progress (Partially Complete) Extra Components
  • Designing and implementing new components and data structures in Java, held to the same standards as Ohio State's Software Components, using the existing components as a base for the new ones.
  • Continuing use of design-by-contract and precise mathematical definitions to describe components' mathematical models.
  • Current components (more to be added later):
    • Deque<T> (complete) - A generic double-ended queue component, allowing the user to access, add elements to, and remove elements from the front and back of the Deque. Various implementations are available:
      • Deque1L - Represented as a java.util.List
      • Deque2 - Represented as a Sequence
      • Deque3 - Represented as a pair of Stacks
      • Deque4 - Represented as a doubly-linked list, done "bare-handed"
    • SortingMachine<T> (complete) - Various new implementations using different structures and sorting algorithms:
      • SortingMachine6 - Represented as a Queue (using an embedding of mergesort)
      • SortingMachine7 - Represented as a Sequence (using an embedding of selection sort)
      • SortingMachine7a - Represented as a Sequence (using an embedding of insertion sort and binary-search)
    • Graph<T> (in progress) - A generic graph component, to be represented as an adjacency-list (represented as a Sequence<Sequence>).
    • Multiset<T> (in progress) - A generic multiset component. It functions like a Set but can support duplicate elements. Two implementations will be available:
      • Multiset1 - Represented as a Map<T, Integer>
      • Multiset2 - Represented as a Sequence
    • Queue<T> (planned) - Two new implementations using a dynamic array:
      • Queue4 - Represented as an Array
      • Queue4a - Represented as a Java built-in array
  • You can view the GitHub repository for this project here.
